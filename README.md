# Mem Proccess — Memory Monitor for Windows

Описание
-
`Mem Proccess` — утилита для мониторинга использования памяти процессов на Windows. Проект включает как исходный Python-скрипт, так и собранный исполняемый файл (PyInstaller) для простого развёртывания на целевых машинах.

Ключевые возможности
-
- Мониторинг использования памяти процессов в реальном времени
- Конфигурируемые параметры через `mem_proccess_config.json`
- Запуск из исходников (`mem_proccess.py`) или как самостоятельного `.exe` в `dist/`
- Сборка через PyInstaller (файлы `mem_proccess.spec`, `Memory Monitor.spec`)

Быстрый старт
-
1) Запуск из Python (для разработки):

```powershell
python mem_proccess.py
```

2) Запуск готового исполняемого файла (без установки Python):

Откройте `dist\mem_proccess.exe` в Проводнике или используйте `run_admin.bat`, если требуются права администратора.

Конфигурация
-
Файл конфигурации: `mem_proccess_config.json`.
В нём задаются параметры мониторинга (интервалы, список отслеживаемых процессов, пороги оповещений и прочее). Пример структуры конфигурации:

```json
{
  "interval_seconds": 1,
  "watch": ["explorer.exe", "chrome.exe"],
  "alert_threshold_mb": 1024,
  "log_file": "mem.log"
}
```

Отредактируйте значения и перезапустите утилиту.

Сборка
-
Проект содержит PyInstaller-спецификации (`mem_proccess.spec`, `Memory Monitor.spec`). Чтобы пересобрать .exe:

```powershell
pip install -r requirements.txt    # при наличии requirements
pyinstaller --clean --onefile mem_proccess.spec
```

Результат сборки появится в `dist/` (и вспомогательные файлы в `build/`).

Логирование и отладка
-
- Версии и метаданные хранятся в `file_version.txt`.
- Для локальной проверки используйте `example.py`.

Безопасность
-
- Используйте запуск с правами администратора только при необходимости.
- Проверяйте конфигурационные файлы перед развёртыванием.

Структура репозитория (кратко)
-
- `mem_proccess.py` — основной скрипт
- `mem_proccess_config.json` — конфигурация
- `mem_proccess.spec`, `Memory Monitor.spec` — PyInstaller спецификации
- `dist/`, `build/` — артефакты сборки
- `run_admin.bat` — запуск с правами администратора
